Google Trends Data Structure

CSE 274: Data Abstraction and Data Structures - Spring 2012

The purpose of this project was to develop a data structure that could be used in a similar context as Google Trends. The program that the data structure is used with goes through a text file and inputs each word into the data structure. The data structure must store all of these words.

The data structure needed to be able to do four functions:

  • increaseCount - adds either a new word to the data structure or, if the word is already present, increases the number of occurences of the word.
  • getCount - returns the current count for a given term
  • getNthPopular - returns the term that is the Nth most occuring word stored in the data structure
  • numEntries - returns the number of terms that are currently stored in the data structure

Each student is ranked for each individual function. Nearly every student tied on getCount and numEntries. I placed first for getNthPopular by sacrificing some time for increaseCount.

This data structure implementation stores the terms in a hash table that uses chaining for collision resolution. In addition, each term also keeps track of the term that is 1 position more popular and 1 position less popular. Each time increaseCount is called the term is checked to see if it should move forward in popularity by comparing it to the next more popular term. This method proved very effective at keeping both getNthPopular and increaseCount relatively fast, until it came time to sort ties alphabetically. At this point, increaseCount became incredibly slow, but getNthPopular was so fast that this method was kept.

Krencrypt Password Manager

Free time - Spring 2012

This project was done for the purpose of managing my own personal passwords. I had been reading quite a bit about information security and entropy at the time and wanted a way to save secure passwords. Rather than have a file containing all of the passwords that could possibly stolen, I decided to make a program that would consistently generate unique passwords.

The program began as a simple command line interface in which the user would provide 3 parameters: the username, the service name, and a master password. Given the same three inputs, the program would always generate the same password, thus allowing the user to only remember the username and a single password. The program quickly grew into a GUI that would accept a key file that can be stored on a hard drive or flash drive, rather than remembering a simple passphrase.

Simple Image Creator

This program is a simple image creator. The user can draw pictures in a manner similar to MS Paint.

    Features in this program include:

  • Change pen size and color
